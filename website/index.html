Patients Like Me

A clinical data analysis platform built on MIMIC-IV demo data. Patient events are converted to RDF, loaded into a QLever triplestore with medical ontologies (ICD9/10, RXNORM, SNOMED CT, LOINC), and explored through interactive visualizations and vector-based patient similarity.

Example SPARQL Queries

Run these against the QLever endpoint or paste into the QLever UI.

Count events per patient (top 10)

PREFIX meds: <https://teamheka.github.io/meds-ontology#>
SELECT ?subject (COUNT(?e) AS ?eventCount) WHERE {
  ?e a meds:Event ; meds:hasSubject ?subject .
}
GROUP BY ?subject
ORDER BY DESC(?eventCount)
LIMIT 10

Get all diagnoses for a patient (with ICD labels)

PREFIX meds: <https://teamheka.github.io/meds-ontology#>
PREFIX skos: <http://www.w3.org/2004/02/skos/core#>
SELECT ?time ?codeString ?label WHERE {
  ?e a meds:Event ;
     meds:hasSubject <https://teamheka.github.io/meds-data/subject/10000032> ;
     meds:codeString ?codeString ;
     meds:time ?time ;
     meds:hasCode ?code .
  ?code meds:parentCode ?parent .
  ?parent skos:prefLabel ?label .
  FILTER(STRSTARTS(?codeString, "ICD"))
}
ORDER BY ?time

Find patients with a specific ICD-9 diagnosis code

PREFIX meds: <https://teamheka.github.io/meds-ontology#>
PREFIX icd9: <http://purl.bioontology.org/ontology/ICD9CM/>
SELECT DISTINCT ?subject WHERE {
  ?e a meds:Event ;
     meds:hasSubject ?subject ;
     meds:hasCode [ meds:parentCode icd9:428.0 ] .
}
